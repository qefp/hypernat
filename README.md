## 使用注意事项

当然，作为NAT VPS，也有一些局限性需要提前了解：

### 什么是NAT VPS？

简单说，NAT VPS就是多个用户共享一个公网IP，通过不同端口区分。这意味着：

- 不能直接使用80/443等常见端口
- 需要通过端口转发访问服务
- 部分需要独立IP的应用可能无法使用

但换个角度看，正因为共享IP，成本才能压得这么低。对于大多数个人用户来说，这些限制其实影响不大。

### IP问题

有用户反馈过，NAT VPS的共享IP有时候会"脏"（被其他用户滥用后进入黑名单）。但 HyperNat 的自动换IP功能在一定程度上缓解了这个问题。如果真遇到IP问题，联系客服可以手动更换。

### 性能预期

虽然硬件配置不错，但毕竟是NAT机，不要拿来跑高负载业务。如果你的项目每天有几万访问量，还是建议上独立IP的VPS或者云服务器。
